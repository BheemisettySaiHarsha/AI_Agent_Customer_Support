# AI-Powered Agentic Customer Support System

**Capstone Project — Milestone 1: Knowledge Base Ingestion & Core Tools**

An agentic customer support system built with **LangChain** and **LangGraph**, following the **ReAct (Reasoning + Acting)** paradigm. The full system will understand customer intent, autonomously select tools, search product documentation, look up orders, analyze sales data, and escalate unresolved issues to a human agent.

This milestone covers the foundation: environment setup, data ingestion, a vector store for the knowledge base, and the first three tools.

## Tech Stack

| Component | Choice |
|---|---|
| LLM | OpenAI `gpt-4o-mini` |
| Embeddings | OpenAI `text-embedding-3-small` |
| Vector Store | Chroma |
| Orchestration (upcoming) | LangChain + LangGraph (ReAct agent) |
| Data handling | Pandas |

## What's in Milestone 1

1. **Environment setup** — dependencies, `.env`-based API key loading, dataset inspection.
2. **Knowledge base ingestion** — the three `.docx` files in `data/knowledge_base/` are loaded, chunked with `RecursiveCharacterTextSplitter`, embedded, and stored in a persistent Chroma collection. Retrieval is verified against sample queries.
3. **Three tools**, implemented as LangChain `@tool` functions so they can be handed directly to a LangGraph agent in Milestone 2:
   - **`knowledge_base_search`** — semantic search over product manuals, FAQs, and support policies.
   - **`order_lookup`** — looks up order details by `order_id` (primary, unique key) or by `email` (returns all orders for that customer).
   - **`sales_analytics`** — Pandas-based aggregation over `sales_data.csv`: revenue, profit, or units sold, grouped by product, category, or region, with optional date-range filtering.

## Setup

**1. Clone the repo and create a virtual environment**
```bash
git clone https://github.com/BheemisettySaiHarsha/AI_Agent_Customer_Support.git
cd AI_Agent_Customer_Support
python -m venv venv
source venv/Scripts/activate     # Git Bash on Windows
# venv\Scripts\activate.bat      # Windows CMD
# source venv/bin/activate       # macOS/Linux
```

**2. Install dependencies**
```bash
pip install -r requirements.txt
```

**3. Configure your API key**
```bash
cp .env.example .env
```
Then edit `.env` and add your key:
```
OPENAI_API_KEY=sk-...
```

**4. Run the notebook**
Open `Milestone1_GenAI_Capstone.ipynb` in VS Code or Jupyter and run all cells in order. This will:
- Load and inspect `orders.csv` and `sales_data.csv`
- Chunk and embed the knowledge base
- Build/persist the Chroma vector store (`chroma_db/`, created on first run)
- Verify retrieval and test all three tools

## Datasets

- **`orders.csv`** — 101 customer orders (`Order_ID`, `Customer_Name`, `Email`, `Product`, `Quantity`, `Order_Date`, `Delivery_Date`, `Status`, `Payment_Method`, `Amount`).
- **`sales_data.csv`** — 900 daily sales records across 5 product categories and 4 regions, Jan–Jun 2025 (`Date`, `Product`, `Category`, `Region`, `Units_Sold`, `Revenue`, `Profit`).
- **`knowledge_base/`** — FAQ, support policies, and a product manual used for retrieval-augmented answers.


## Notes

- The Chroma vector store (`chroma_db/`) is a build artifact regenerated by rerunning the ingestion cells — it's excluded from version control.
- Never commit `.env`. Use `.env.example` as the template for required variables.
